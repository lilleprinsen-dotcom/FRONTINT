# Agent Instructions

- Prefer batching related local changes into one larger work batch to reduce overhead and repeated verification cost.
- For every code or documentation edit batch, create a new branch and a new pull request instead of updating an old PR.
- Do not open tiny PRs for every small subtask when the changes belong to the same requested goal.
- Run the relevant local verification once per completed batch before pushing.
- Push, open, and merge the new PR when the batch is complete and checks pass.
