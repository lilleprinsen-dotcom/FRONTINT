# Front Product Write Contract

This contract is based on the local Front Systems OpenAPI file:

`docs/vendor/front-systems/openapi/frontsystems.openapi.json`

## Chosen Endpoint For First Limited Test

Use the single-product create endpoint:

`POST /api/products`

Reasons:

- It is the documented product CRUD create endpoint.
- It accepts one product payload at a time, which is safer for the first real write test than bulk endpoints.
- It returns a product payload that can be used to capture Front product identifiers per selected item.
- It supports `extId`, which can be used as the stable external identifier for future updates.

Do not use these endpoints in the first limited write test:

- `POST /api/Product`: read-only product listing/search in Front's OpenAPI.
- `PUT /api/products/{productId}`: update path; use later after successful create/update identity behavior is confirmed.
- `POST /api/v2/Products/bulk-insert`: bulk path; use later for controlled batch sync.
- `PUT /api/v2/Products/bulk-upsert`: bulk path; use later after single-product behavior is proven.
- `POST /api/PricelistV2`: price-list path; sale price handling comes later.
- `POST /api/Stock/adjust`: stock writes are not part of product write testing.

## Authentication

Header:

`x-api-key: <Front Systems API key>`

The API key must come from encrypted connection credential storage. It must never be logged or displayed.

## Request Schema

The OpenAPI request body references:

`WebshopAPI.ViewModels.WebUpdateableProduct`

Important fields used by OmniBridge for the first limited test:

- `createProductSpecificSize`
- `extId`
- `name`
- `number`
- `variant`
- `groupName`
- `subgroupName`
- `color`
- `season`
- `brand`
- `price`
- `isStockProduct`
- `isWebAvailable`
- `productSizes`
- `images`
- `isNoLabel`

`productSizes[]` supports:

- `gtin`
- `label`
- `externalSKU`
- `labelSortIndex`

## Identifier Strategy

For the first limited write test:

- Woo item key remains the internal source identity.
- Front `extId` is generated from Woo item key:
  - `product:123` -> `woo-product-123`
  - `variation:456` -> `woo-variation-456`
- Woo SKU maps to Front `number`.
- Variation size label maps to Front `variant` and `productSizes[].label`.
- Woo SKU maps to `productSizes[].externalSKU`.
- Woo GTIN/EAN maps to `productSizes[].gtin` when available.

NEEDS_FRONT_CONFIRMATION:

- Whether `number` should be parent style number instead of variation SKU for variable products.
- Whether `variant` should be size label, color code, or another Front-specific value.
- Whether `extId` accepts arbitrary strings with hyphens and whether it is unique across all products.
- Whether `productSizes[].identity` is generated by Front or can/should be supplied by OmniBridge.
- Whether SKU-only items without GTIN/EAN are acceptable for POS workflows.

## Required Fields And Placeholders

The OpenAPI schema marks `name`, `brand`, `groupName`, `season`, and `color` as required, while several of those properties are nullable.

For the first limited test:

- `name` must be present.
- `brand`, `groupName`, `subgroupName`, `color`, and `season` are sent as available from the preview payload.
- Missing `color` and `season` are sent as `null`.

NEEDS_FRONT_CONFIRMATION:

- Whether nullable-but-required fields may be sent as `null`.
- Whether Front auto-creates `brand`, `groupName`, `subgroupName`, `season`, and `color`.
- Whether missing brand/category should block the first write test for Lilleprinsen.

## Response Shape

The OpenAPI response is HTTP `200` with:

`WebshopAPI.ViewModels.WebUpdateableProduct`

Relevant response fields:

- `id`
- `extId`
- `productid`
- `number`
- `variant`
- `productSizes[].identity`
- `productSizes[].gtin`
- `productSizes[].externalSKU`

OmniBridge stores only a sanitized response summary:

- HTTP status
- Front product id
- Front ext id
- Front identity
- Front external SKU
- selected safe response keys

No secrets or full raw response bodies should be logged.

## Safety Gates

The first limited write test must only run when:

- Product sync profile mode is `limited_write_test`.
- `OMNIBRIDGE_ALLOW_PRODUCTION_WRITES=false`.
- A Front Systems connection exists.
- Front API key is configured.
- The selected items are from a preview run.
- At most 10 items are selected.
- Selected items are `ready` or `warning`.
- Blocked items are rejected.

The first limited write test must not:

- Write to WooCommerce.
- Write prices through `PricelistV2`.
- Write stock.
- Import orders.
- Trigger full catalog sync.
